# Join이란?

실무에서는 하나의 데이터만 사용하는 경우보다 **여러 개의 데이터셋을 결합하여 분석하는 경우가 훨씬 많다.**

예를 들어 다음과 같은 데이터가 있다고 가정해보자.

**회원 정보**

| ID | 이름 |
|---|---|
| A | 김철수 |
| B | 이영희 |
| C | 박민수 |

**주문 정보**

| ID | 주문금액 |
|---|---|
| A | 30000 |
| C | 15000 |
| D | 50000 |

회원 이름과 주문 금액을 함께 분석하려면 두 데이터를 하나로 합쳐야 한다.

이처럼 **공통된 열(Key)을 기준으로 여러 데이터를 결합하는 작업을 Join이라고 한다.**

---

# dplyr Join 함수

dplyr에서는 대표적으로 네 가지 Join 함수를 제공한다.

| 함수 | 설명 |
|------|------|
| left_join() | 왼쪽 데이터 기준으로 결합 |
| right_join() | 오른쪽 데이터 기준으로 결합 |
| inner_join() | 공통 데이터만 결합 |
| full_join() | 모든 데이터를 유지 |

SQL의 JOIN과 동일한 개념이다.

---

# 실습 데이터 생성

```r
library(dplyr)

member <- data.frame(
  ID = c("A", "B", "C", "D"),
  Name = c("Kim", "Lee", "Park", "Choi")
)

order <- data.frame(
  ID = c("A", "B", "C", "E"),
  Price = c(30000, 45000, 12000, 50000)
)
```

회원 정보

|ID|Name|
|---|---|
|A|Kim|
|B|Lee|
|C|Park|
|D|Choi|

주문 정보

|ID|Price|
|---|---|
|A|30000|
|B|45000|
|C|12000|
|E|50000|

---

# Key란?

Join에서 가장 중요한 개념은 **Key**이다.

Key는 두 데이터에서 서로 같은 데이터를 찾기 위한 기준 열이다.

위 예제에서는 **ID**가 Key이다.

```r
left_join(member, order, by = "ID")
```

`by = "ID"`는 ID 열을 기준으로 데이터를 결합한다는 의미이다.

---

# left_join()

## 개념

왼쪽 데이터의 모든 행을 유지하면서 오른쪽 데이터를 결합한다.

일치하지 않는 데이터는 **NA**가 된다.

가장 많이 사용하는 Join 방식이다.

---

## 예제

```r
left_join(member, order, by = "ID")
```

결과

|ID|Name|Price|
|---|---|---|
|A|Kim|30000|
|B|Lee|45000|
|C|Park|12000|
|D|Choi|NA|

회원 D는 주문 정보가 없기 때문에 Price가 NA로 표시된다.

반면 주문 정보에만 있는 E는 제외된다.

---

# right_join()

## 개념

오른쪽 데이터의 모든 행을 유지한다.

왼쪽 데이터에 없는 값은 NA가 된다.

---

## 예제

```r
right_join(member, order, by = "ID")
```

결과

|ID|Name|Price|
|---|---|---|
|A|Kim|30000|
|B|Lee|45000|
|C|Park|12000|
|E|NA|50000|

주문 정보는 모두 유지된다.

회원 정보에 없는 E는 Name이 NA가 된다.

---

# inner_join()

## 개념

두 데이터에 모두 존재하는 데이터만 가져온다.

공통 데이터만 분석할 때 사용한다.

---

## 예제

```r
inner_join(member, order, by = "ID")
```

결과

|ID|Name|Price|
|---|---|---|
|A|Kim|30000|
|B|Lee|45000|
|C|Park|12000|

D와 E는 제외된다.

---

# full_join()

## 개념

두 데이터의 모든 행을 유지한다.

일치하지 않는 값은 NA가 된다.

---

## 예제

```r
full_join(member, order, by = "ID")
```

결과

|ID|Name|Price|
|---|---|---|
|A|Kim|30000|
|B|Lee|45000|
|C|Park|12000|
|D|Choi|NA|
|E|NA|50000|

데이터가 하나도 손실되지 않는다.

---

# 여러 개의 Key 사용하기

실무에서는 하나의 Key만 사용하는 경우보다 여러 개의 Key를 함께 사용하는 경우도 많다.

예를 들어 고객 ID와 주문 연도가 모두 같을 때만 데이터를 결합한다고 가정해보자.

```r
sales <- data.frame(
  ID = c("A","A","B","B"),
  Year = c(2024,2025,2024,2025),
  Sales = c(100,120,80,90)
)

profit <- data.frame(
  ID = c("A","A","B","B"),
  Year = c(2024,2025,2024,2025),
  Profit = c(20,30,15,18)
)
```

Join 실행

```r
left_join(sales,
          profit,
          by = c("ID","Year"))
```

결과

|ID|Year|Sales|Profit|
|---|---|---|---|
|A|2024|100|20|
|A|2025|120|30|
|B|2024|80|15|
|B|2025|90|18|

`by = c("ID", "Year")`는 두 개의 열이 모두 일치하는 경우에만 데이터를 결합한다.

---

# Join 시 주의사항

## 1. Key 값이 중복되면 행이 늘어날 수 있다.

예를 들어 하나의 회원이 주문을 여러 번 했다면 Join 결과의 행 수가 증가한다.

분석 전에 Key가 고유한 값인지 반드시 확인하는 것이 좋다.

---

## 2. Key의 자료형이 같아야 한다.

예를 들어 한쪽은 숫자(integer), 다른 한쪽은 문자(character)라면 Join이 정상적으로 수행되지 않는다.

```r
str(member)
str(order)
```

자료형을 먼저 확인하는 습관을 들이는 것이 좋다.

---

## 3. 결측치(NA)를 확인한다.

Join 이후에는 반드시 NA가 생성되었는지 확인해야 한다.

```r
sum(is.na(result))
```

예상보다 많은 NA가 발생했다면 Key가 잘못 지정되었을 가능성이 있다.

# 마무리

이번 튜토리얼에서는 **dplyr의 Join 함수**를 학습하였다.

Join은 여러 데이터셋을 하나로 결합하는 핵심 기능이며, 데이터 분석과 머신러닝 프로젝트에서 매우 자주 사용된다.

특히 다음 네 가지 Join의 차이를 반드시 이해해야 한다.

- **right_join()** : 오른쪽 데이터를 모두 유지한다.
- **inner_join()** : 공통 데이터만 유지한다.
- **full_join()** : 모든 데이터를 유지한다.
