# 결측치(NA)의 처리

## 결측치(NA)란?

결측치(NA, Missing Value)는 데이터에 값이 존재하지 않는 경우를 의미한다.

데이터 분석에서는 결측치를 그대로 사용하면 평균, 합계, 회귀분석 등의 계산 결과가 왜곡될 수 있다. 따라서 분석을 수행하기 전에 결측치를 확인하고 적절한 방법으로 처리하는 과정이 필요하다.

결측치를 처리하는 대표적인 방법은 다음과 같다.

- 결측치가 포함된 행 제거
- 평균(mean)으로 대체
- 중앙값(median)으로 대체
- 특정 값(0 등)으로 대체

R에서는 `na.omit()`, `na.rm`, `is.na()`, `anyNA()`, `mutate()` 등의 함수를 이용하여 결측치를 쉽게 처리할 수 있다.

```r
PATH <- "https://raw.githubusercontent.com/guru99-edu/R-Programming/master/test.csv"
```

# 결측치가 있는 열 확인하기

데이터 프레임에서 어떤 열에 결측치가 존재하는지 확인할 수 있다.

```r
list_na <- colnames(df_titanic)[
  apply(df_titanic, 2, anyNA)
]

list_na
```

### 결과

```text
[1] "Age" "Fare"
```

### 코드 설명

```r
apply(df_titanic, 2, anyNA)
```

- `2`는 열(column)을 의미한다.
- `anyNA()`는 해당 열에 NA가 하나라도 있는지 확인한다.

다음 코드

```r
colnames(df_titanic)[apply(df_titanic, 2, anyNA)]
```

는 결측치가 존재하는 열 이름만 반환한다.

# na.rm 옵션

평균이나 합계를 계산할 때 결측치가 포함되어 있으면 결과도 NA가 된다.

```r
score <- c(80, 90, NA, 70)

mean(score)
```

### 결과

```text
[1] NA
```

결측치를 제외하고 계산하려면 `na.rm = TRUE`를 사용한다.

```r
mean(score, na.rm = TRUE)
```

### 결과

```text
[1] 80
```

`na.rm`은 **NA를 제거(remove)하고 계산한다**는 의미이다.

# 평균으로 결측치 대체하기

결측치를 제거하는 대신 평균값으로 대체하는 방법도 자주 사용된다.

먼저 결측치가 있는 열의 평균을 계산한다.

```r
average_missing <- apply(

  df_titanic[, colnames(df_titanic) %in% list_na],

  2,

  mean,

  na.rm = TRUE

)

average_missing
```

### 결과

```text
      Age     Fare
30.27259 35.62719
```

### 코드 설명

`apply()` 함수는 여러 열에 동일한 함수를 적용할 때 사용한다.

```r
apply(
  데이터,
  2,
  mean,
  na.rm = TRUE
)
```

- 첫 번째 인수 : 평균을 계산할 데이터
- `2` : 열(column) 기준 계산
- `mean` : 평균 계산 함수
- `na.rm = TRUE` : 결측치를 제외하고 계산

# mutate()를 이용하여 평균으로 대체

평균을 계산한 후 `mutate()`와 `ifelse()`를 이용하여 새로운 변수를 생성한다.

```r
library(dplyr)

df_replace <- df_titanic %>%

  mutate(

    replace_mean_age =
      ifelse(
        is.na(Age),
        average_missing["Age"],
        Age
      ),

    replace_mean_fare =
      ifelse(
        is.na(Fare),
        average_missing["Fare"],
        Fare
      )

  )
```

위 코드는 원본 데이터를 변경하지 않고 새로운 변수를 생성한다.

# 중앙값으로 결측치 대체하기

평균 대신 중앙값(Median)을 사용하는 경우도 많다.

특히 이상치(Outlier)가 많은 데이터에서는 평균보다 중앙값을 사용하는 것이 안정적이다.

먼저 중앙값을 계산한다.

```r
median_missing <- apply(

  df_titanic[, colnames(df_titanic) %in% list_na],

  2,

  median,

  na.rm = TRUE

)

median_missing
```

### 결과

```text

```

```r
df_replace <- df_titanic %>%

      ifelse(
        is.na(Fare),
        median_missing["Fare"],
        Fare
      )

  )
```

# 평균과 중앙값의 차이

| 방법 | 특징 |
|------|------|
| 평균(Mean) | 모든 데이터를 이용하여 계산한다. 이상치의 영향을 많이 받는다. |
| 중앙값(Median) | 가운데 값을 사용한다. 이상치의 영향을 거의 받지 않는다. |

예를 들어 다음 데이터를 살펴보자.

```r
x <- c(10, 20, 30, 40, 500)
```

평균

```r
mean(x)
```

결과

```text
[1] 120
```

중앙값

```r
median(x)
```

결과

```text
[1] 30
```

500이라는 이상치 때문에 평균은 크게 증가하지만, 중앙값은 거의 영향을 받지 않는다.

실무에서는 이상치가 많다면 평균보다 중앙값을 사용하는 경우가 많다.

# sapply()를 이용한 결측치 일괄 처리

데이터의 열이 많다면 하나씩 평균을 계산하여 대체하는 작업은 번거롭다.

이 경우 `sapply()`를 이용하면 모든 열을 한 번에 처리할 수 있다.

```r
df_impute_mean <- data.frame(

  sapply(

    df_titanic,

    function(x)

      ifelse(

        is.na(x),

        mean(x, na.rm = TRUE),

        x

      )

  )

)
```

위 코드는 모든 변수에 대해 다음 작업을 수행한다.

1. 결측치 여부 확인
2. 평균 계산
3. 결측치를 평균으로 대체
4. 새로운 데이터 프레임 생성

---

# sapply() 코드 분석

```r
sapply(

  df_titanic,

  function(x){

    ifelse(
      is.na(x),
      mean(x, na.rm = TRUE),
      x
    )

  }

)
```

각 요소의 의미는 다음과 같다.

- `df_titanic` : 모든 열을 대상으로 반복
- `function(x)` : 각 열을 하나씩 처리
- `is.na(x)` : 결측치 확인
- `mean(x, na.rm = TRUE)` : 평균 계산
- `ifelse()` : 결측치만 평균으로 변경

# lapply()를 이용한 평균 대체

`lapply()`를 이용하면 여러 열의 결측치를 한 번에 처리할 수 있다.

```r
df <- data.frame(

  math = c(90, NA, 80, 70),

  english = c(80, 75, NA, 95)

)

df[] <- lapply(

  df,

  function(x){

    x[is.na(x)] <- mean(x, na.rm = TRUE)

    x

  }

)

df
```

### 결과

```text

1   90       80
2   80       75
3   80       83.3
4   70       95
```

`lapply()`는 데이터 프레임의 여러 열을 동시에 처리할 때 자주 사용된다.

# 결측치 처리 방법 비교

| 방법 | 특징 | 장점 | 단점 |
|------|------|------|------|
| `na.omit()` | 결측치 제거 | 가장 간단함 | 데이터 손실 발생 |
| 평균 대체 | 평균으로 변경 | 계산이 쉬움 | 이상치의 영향을 받음 |
| 중앙값 대체 | 중앙값으로 변경 | 이상치에 강함 | 데이터 분포가 달라질 수 있음 |
| 특정 값 대체 | 원하는 값 사용 | 상황에 따라 유용 | 임의성이 존재 |

| `sapply()` | 여러 변수에 동일한 작업 수행 |

# 정리

결측치는 데이터 분석에서 반드시 확인하고 처리해야 하는 요소이다.

- `is.na()`와 `anyNA()`는 결측치를 확인하는 함수이다.
- `na.omit()`은 결측치가 포함된 행을 제거한다.
- `na.rm = TRUE`는 계산 시 결측치를 제외한다.
- `mutate()`와 `ifelse()`를 이용하면 평균이나 중앙값으로 결측치를 대체할 수 있다.
- `apply()`는 여러 열의 평균이나 중앙값을 계산할 때 사용한다.
- `lapply()`와 `sapply()`는 여러 변수의 결측치를 한 번에 처리할 때 유용하다.
- `complete.cases()`는 결측치가 없는 행만 선택할 수 있다.

결측치를 어떤 방법으로 처리할지는 데이터의 특성과 분석 목적에 따라 달라진다. 단순히 삭제하기보다 평균, 중앙값 또는 적절한 대체값을 사용하는 것이 더 좋은 결과를 얻는 경우도 많다.
