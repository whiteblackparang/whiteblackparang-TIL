# Condition

조건문(Condition)은 조건의 결과(True 또는 False)에 따라 서로 다른 코드를 실행하는 문법이다. 프로그램의 흐름을 제어할 때 가장 많이 사용된다.

---

## if

- 조건이 `True`일 때만 코드를 실행한다.
- 조건 뒤에는 반드시 `:`을 작성한다.
- 실행할 코드는 들여쓰기로 구분한다.
- 조건이 `False`이면 if 블록은 실행되지 않는다.

---

## if-else

- 조건에 따라 두 가지 경우 중 하나를 실행한다.
- 조건이 `True`이면 `if` 블록을 실행한다.
- 조건이 `False`이면 `else` 블록을 실행한다.
- 둘 중 하나는 반드시 실행된다.

---

## if-elif-else

- 여러 조건을 순서대로 검사한다.
- 처음으로 `True`가 되는 조건만 실행된다.
- `elif`는 여러 개 사용할 수 있다.
- `else`는 모든 조건이 거짓일 때 실행된다.

---

## 비교 연산자

조건문에서 가장 많이 사용하는 연산자이다.

| 연산자 | 의미 |
|--------|------|
| `==` | 같다 |
| `!=` | 같지 않다 |
| `>` | 크다 |
| `<` | 작다 |
| `>=` | 크거나 같다 |
| `<=` | 작거나 같다 |

- 비교 결과는 `True` 또는 `False`이다.
- 숫자뿐 아니라 문자열도 비교할 수 있다.

---

## 논리 연산자

### and

- 모든 조건이 `True`여야 `True`를 반환한다.
- 여러 조건을 동시에 만족해야 할 때 사용한다.

### or

- 하나의 조건만 `True`여도 `True`를 반환한다.
- 여러 조건 중 하나만 만족하면 될 때 사용한다.

### not

- 조건의 결과를 반대로 바꾼다.
- `True`는 `False`, `False`는 `True`가 된다.

---

## in / not in

- 값이 컬렉션에 포함되어 있는지 확인한다.
- 리스트, 문자열, 튜플, 집합, 딕셔너리에서 사용할 수 있다.
- `not in`은 포함되어 있지 않은지 검사한다.

---

## 중첩 조건문

- 조건문 안에 또 다른 조건문을 작성할 수 있다.
- 여러 단계를 거쳐 조건을 검사할 때 사용한다.
- 중첩이 너무 많으면 코드가 복잡해질 수 있으므로 `and`, `or`를 사용하는 것이 더 읽기 쉬운 경우도 있다.

---

## 삼항 연산자

- 간단한 `if-else`를 한 줄로 작성하는 방법이다.
- 형식은 다음과 같다.

```python
값1 if 조건 else 값2
```

- 조건이 `True`이면 앞의 값을 반환한다.
- 조건이 `False`이면 뒤의 값을 반환한다.
- 간단한 조건 분기에 사용하는 것이 좋다.

---

# 조건문 작성 규칙

- 조건 뒤에는 반드시 `:`을 작성한다.
- 실행할 코드는 같은 들여쓰기 수준을 유지한다.
- 위에서 아래 순서대로 조건을 검사한다.
- 가장 먼저 `True`가 된 조건만 실행된다.
- 복잡한 조건은 논리 연산자를 활용하면 코드가 간결해진다.

---

# 자주 사용하는 조건문 형태

| 문법 | 설명 |
|------|------|
| `if` | 조건이 참일 때 실행 |
| `if-else` | 두 가지 경우 분기 |
| `if-elif-else` | 여러 조건 분기 |
| `and` | 모두 만족 |
| `or` | 하나 이상 만족 |
| `not` | 조건 반전 |
| `in` | 포함 여부 검사 |
| `not in` | 미포함 여부 검사 |
| `삼항 연산자` | 한 줄 조건문 |

---

# 핵심 정리

- 조건문의 결과는 항상 `True` 또는 `False`이다.
- `if`는 조건이 참일 때만 실행된다.
- `if-else`는 둘 중 하나를 반드시 실행한다.
- `if-elif-else`는 여러 조건을 순서대로 검사한다.
- `and`, `or`, `not`으로 여러 조건을 조합할 수 있다.
- `in`과 `not in`은 포함 여부를 검사한다.
- 간단한 조건은 삼항 연산자로 표현할 수 있다.
