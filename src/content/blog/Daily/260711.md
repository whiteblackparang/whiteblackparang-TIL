# 오늘 배운 점

파이썬 공식 문서 4장 앞부분(제어 흐름 도구)을 읽고 정리한다. `if`, `for`, `range()`, `break`/`continue`, 반복문의 `else` 절, `pass`, `match` 문을 다룬다.

## 1. `if` / `elif` / `else` — `elif`는 들여쓰기를 줄여주는 문법이다

- `elif`는 `else if`의 줄임말이다. 조건을 여러 개 연달아 검사할 때, `else` 안에 또 `if`를 넣는 대신 `elif`를 쓰면 들여쓰기가 계속 깊어지는 것을 막을 수 있다.
- `elif`는 개수 제한이 없고, `else`는 있어도 되고 없어도 된다.
- 여러 언어의 `switch`/`case` 문이 하는 역할을 파이썬에서는 `if … elif … elif …` 흐름이 대신한다.

```python
grade = 85

if grade >= 90:
    label = 'A'
elif grade >= 80:
    label = 'B'
elif grade >= 70:
    label = 'C'
else:
    label = 'F'

label
```

`grade`가 `85`이므로 첫 번째 조건(`>= 90`)은 거짓, 두 번째 조건(`>= 80`)이 참이라 `label`은 `'B'`가 된다. 이 조건이 참으로 확인되는 순간 나머지 `elif`, `else`는 검사하지 않고 건너뛴다.

## 2. `for` 문은 숫자가 아니라 시퀀스의 항목을 직접 순회한다

- C나 파스칼의 `for`문과 다르게, 파이썬의 `for`문은 리스트나 문자열 같은 시퀀스에 들어있는 항목을 순서대로 하나씩 꺼내서 순회한다. "0부터 몇까지 반복하라"는 방식이 아니다.

```python
cities = ['Seoul', 'Busan', 'Incheon']
for city in cities:
    print(city, len(city))
```

`cities`에 담긴 문자열을 순서대로 하나씩 꺼내 `city`에 대입하면서 반복하고, 각 도시 이름과 그 길이를 출력한다.

## 3. 순회 중인 컬렉션 자체를 수정하면 안 된다

- `for` 문으로 딕셔너리나 리스트를 순회하면서 동시에 그 컬렉션의 항목을 지우거나 추가하면 예상치 못한 동작이 생기기 쉽다.
- 안전한 방법은 두 가지다: (1) 원본의 사본을 순회하면서 원본을 수정하거나, (2) 아예 새 컬렉션을 만들어서 조건에 맞는 항목만 옮겨 담는 것이다.

```python
inventory = {'pen': 0, 'notebook': 5, 'eraser': 0}

# 전략 1: 사본을 순회하면서 원본에서 삭제한다
for item, qty in inventory.copy().items():
    if qty == 0:
        del inventory[item]

# 전략 2: 새 딕셔너리를 만든다
stockedItems = {}
for item, qty in inventory.items():
    if qty > 0:
        stockedItems[item] = qty
```

## 4. `range()`는 숫자 시퀀스를 만드는 내장 함수다

- `range(n)`은 `0`부터 `n-1`까지의 값을 만든다. 끝값(`n`)은 결과에 포함되지 않는다.
- 시작값, 끝값, 증가폭(스텝)을 지정할 수 있고, 스텝은 음수도 가능하다.

```python
list(range(3, 8))          # [3, 4, 5, 6, 7]
list(range(0, 12, 4))      # [0, 4, 8]
list(range(10, 0, -2))     # [10, 8, 6, 4, 2]
```

- 인덱스로 시퀀스를 순회하고 싶으면 `range(len(리스트))`와 결합해서 쓸 수 있지만, 항목과 인덱스를 함께 얻고 싶을 때는 `enumerate()`가 더 편리하다.

## 5. `range()`가 돌려주는 것은 리스트가 아니라 이터러블이다

- `range(10)`을 그냥 출력하면 `[0, 1, ..., 9]`가 아니라 `range(0, 10)`이라는 표현이 나온다. 실제 리스트를 미리 만들어두지 않고, 필요할 때마다 다음 값을 계산해서 내놓는 객체이기 때문이다.
- 이런 객체를 이터러블이라고 부른다. `for` 문뿐 아니라 `sum()`처럼 "항목을 순서대로 하나씩 받는" 함수에도 바로 넘길 수 있다.

```python
sum(range(5))   # 0+1+2+3+4 = 10
```

## 6. `break` 문 — 가장 가까운 반복문을 즉시 빠져나간다

`break`가 실행되면, 그 `break`를 감싸고 있는 가장 안쪽의 `for`나 `while` 반복문에서 즉시 빠져나간다.

```python
for number in range(10, 20):
    if number % 7 == 0:
        print(number, '은 7의 배수다')
        break
```

`14`에서 `number % 7 == 0`이 참이 되는 순간 메시지를 출력하고 `break`로 반복문을 즉시 끝낸다. `21`처럼 그 뒤에 있을 다른 7의 배수는 검사하지 않는다.

## 7. `continue` 문 — 이번 회차만 건너뛰고 다음 반복으로 넘어간다

`continue`는 반복문 전체를 끝내지 않고, 현재 회차만 중단하고 바로 다음 반복으로 넘어간다.

```python
for number in range(1, 6):
    if number == 3:
        continue
    print(number)
```

`number`가 `3`일 때만 `print`를 건너뛰고, 나머지 `1, 2, 4, 5`는 정상적으로 출력된다.

## 8. 반복문에도 `else` 절을 붙일 수 있다

- `for`나 `while` 뒤에 `else`를 붙이면, 반복문이 `break`로 중간에 끝나지 않고 끝까지 정상적으로 마쳤을 때만 그 `else` 블록이 실행된다.
- `break`로 중간에 빠져나왔다면 `else`는 실행되지 않는다.
- 반복문의 `else`는 `if`문의 `else`보다는, "예외가 발생하지 않았을 때만 실행되는" `try`문의 `else`와 더 비슷한 개념으로 이해하면 헷갈리지 않는다.

```python
for candidate in range(2, 10):
    for divisor in range(2, candidate):
        if candidate % divisor == 0:
            print(candidate, '=', divisor, '*', candidate // divisor)
            break
    else:
        print(candidate, '는 소수다')
```

안쪽 반복문에서 나누어떨어지는 값을 찾으면 `break`로 빠져나가고, 그 경우 바깥 `for`에 붙은 `else`는 실행되지 않는다. 반대로 끝까지 나누어떨어지는 값을 못 찾았다면(즉 `break`가 한 번도 실행되지 않았다면) `else` 블록이 실행되어 "소수다"라는 메시지가 출력된다.

## 9. `pass` 문 — 문법적으로 문장이 필요하지만 할 일이 없을 때 쓴다

- `pass`는 아무 동작도 하지 않는다. 파이썬 문법상 블록 안에 최소 한 줄이 있어야 하는데, 지금 당장 채울 내용이 없을 때 자리만 채워두는 용도로 쓴다.
- 빈 클래스를 만들 때나, 나중에 구현할 함수의 몸체를 임시로 비워둘 때 자주 쓰인다.

```python
class PlaceholderModel:
    pass

def notImplementedYet(*args):
    pass  # TODO: 나중에 구현
```

## 10. `match` 문 — 값의 모양에 따라 분기하는 패턴 매칭

- `match` 문은 하나의 값을 여러 개의 `case` 패턴과 순서대로 비교해서, 가장 먼저 일치하는 패턴의 블록만 실행한다. 다른 언어의 `switch`와 겉모습은 비슷하지만, 값을 통째로 비교하는 것을 넘어 값의 구조를 분해해서 변수로 꺼낼 수 있다는 점이 다르다.
- `case _:`는 어떤 값과도 매칭되는 와일드카드다. 다른 모든 패턴에 해당하지 않을 때 마지막 안전망 역할을 한다.
- `|`를 쓰면 여러 리터럴을 하나의 패턴으로 묶어서 검사할 수 있다.

```python
def httpMessage(statusCode):
    match statusCode:
        case 200 | 201:
            return 'success'
        case 404:
            return 'not found'
        case 500:
            return 'server error'
        case _:
            return 'unknown status'

httpMessage(201)  # 'success'
```

- 패턴 안에 변수 이름을 두면, 그 값을 변수로 바로 꺼내 쓸 수 있다. 튜플이나 리스트 모양의 패턴은 언패킹 대입과 비슷하게 동작한다.

```python
position = (0, 5)

match position:
    case (0, 0):
        description = 'origin'
    case (0, y):
        description = f'y축 위, y={y}'
    case (x, 0):
        description = f'x축 위, x={x}'
    case (x, y):
        description = f'평면 위 점, x={x}, y={y}'
```

`position`이 `(0, 5)`이므로 `x`가 `0`인 두 번째 패턴 `case (0, y):`에 걸려서, `y`에 `5`가 바인딩되고 `description`은 `'y축 위, y=5'`가 된다.

- 클래스 인스턴스도 패턴으로 검사할 수 있는데, `__match_args__`를 클래스에 정의해두면 `Class(값1, 값2)`처럼 위치 인자만으로도 속성을 매칭할 수 있다.
- 패턴 뒤에 `if 조건`을 붙이면(가드), 패턴이 일치하더라도 조건이 거짓이면 다음 `case`로 넘어간다.

```python
class Coordinate:
    __match_args__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

def describe(point):
    match point:
        case Coordinate(x, y) if x == y:
            return f'대각선 위의 점, {x}=={y}'
        case Coordinate(x, y):
            return '대각선 밖의 점'

describe(Coordinate(3, 3))  # '대각선 위의 점, 3==3'
```
