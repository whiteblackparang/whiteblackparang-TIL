## 오늘 배운 점

파이썬 공식 문서의 "리스트"와 "프로그래밍으로의 첫걸음" 부분을 읽고 정리한다. 리스트 자체는 익숙했지만, "대입은 데이터를 복사하지 않는다"는 규칙과 슬라이스 대입으로 리스트 길이 자체를 바꿀 수 있다는 점은 새로 확인한 내용이다.

### 1. 리스트는 대괄호로 만들고, 인덱싱·슬라이싱은 문자열과 같은 방식으로 동작한다

- 리스트는 대괄호 `[]` 안에 값을 쉼표로 나열해서 만든다. 서로 다른 타입의 값을 섞어 담을 수도 있지만, 실무에서는 보통 같은 타입끼리 담는다.
- 인덱싱(`리스트[i]`)은 항목 하나를 돌려주고, 슬라이싱(`리스트[i:j]`)은 새 리스트를 돌려준다.

```python
temperatures = [18, 21, 19, 25, 30]
temperatures[0]     # 18, 첫 번째 항목
temperatures[-1]    # 30, 마지막 항목
temperatures[-2:]   # [25, 30], 뒤에서 두 개를 새 리스트로 반환
```

### 2. 리스트는 문자열과 달리 가변이다 — 내용을 직접 바꿀 수 있다

- 문자열은 한 번 만들면 내용을 바꿀 수 없는(불변) 자료형이지만, 리스트는 만든 뒤에도 특정 위치의 값을 바꾸거나 새 값을 추가할 수 있다(가변).
- 인덱스로 특정 위치에 값을 다시 대입하면 그 자리의 값만 바뀐다.
- `리스트.append(값)`을 쓰면 리스트 맨 끝에 새 항목이 추가된다.

```python
scores = [70, 85, 90]
scores[0] = 75          # 첫 번째 값을 75로 고친다
scores.append(100)      # 맨 끝에 100을 추가한다
scores
```

`scores[0] = 75`가 실행되면 첫 항목이 `75`로 바뀌고, `scores.append(100)`이 실행되면 리스트 끝에 `100`이 붙어서 최종적으로 `[75, 85, 90, 100]`이 출력된다.

### 3. 리스트를 변수에 대입해도 "복사"가 아니라 "같은 리스트를 같이 가리키는 것"이다

- 파이썬에서 단순 대입(`b = a`)은 데이터를 복사하지 않는다. 리스트를 다른 변수에 대입하면, 두 변수는 서로 다른 리스트가 아니라 같은 리스트 하나를 동시에 가리킨다.
- 그래서 한 변수를 통해 리스트를 바꾸면, 같은 리스트를 가리키는 다른 모든 변수에서도 그 변화가 그대로 보인다.
- 이 사실을 모르고 "그냥 복사해서 따로 쓰는 것"이라고 착각하면, 의도치 않게 원본 리스트까지 바뀌는 버그가 생긴다.

```python
teamA = ['Yuna', 'Minho']
teamB = teamA           # 복사가 아니라 같은 리스트를 같이 가리킨다
id(teamA) == id(teamB)  # True: 두 이름이 가리키는 객체가 완전히 같다

teamB.append('Suji')
teamA                   # ['Yuna', 'Minho', 'Suji'] -- teamA도 함께 바뀐다
```

### 4. 진짜 복사본이 필요하면 슬라이스로 만들어야 한다

- 슬라이스 연산(`리스트[:]` 등)은 요청한 항목들을 담은 **새 리스트**를 돌려준다. 그래서 `리스트[:]`로 만든 결과는 원본과는 별개의 리스트다(단, 리스트 안에 또 다른 가변 객체가 들어 있다면 그 내부 객체까지 완전히 복사되는 것은 아니라서, 이런 방식을 "얕은 복사"라고 부른다).
- "같은 리스트를 같이 가리키는 것"과 "슬라이스로 만든 새 리스트"의 차이를 구분하는 것이 중요하다.

```python
original = ['A', 'B', 'C']
sameRef = original       # 같은 리스트를 같이 가리킨다
copiedList = original[:]  # 새 리스트(얕은 복사본)를 만든다

copiedList[0] = 'Z'
original     # ['A', 'B', 'C'] -- 원본은 그대로다
sameRef       # ['A', 'B', 'C'] -- 이것도 원본과 같은 객체라 그대로다
copiedList    # ['Z', 'B', 'C'] -- 복사본만 바뀌었다
```

### 5. 슬라이스에 값을 대입하면 리스트의 길이 자체를 바꿀 수 있다

- 슬라이스 위치에 새 값을 대입하는 것도 가능하다. 이때 대입하는 값의 개수가 원래 범위의 길이와 달라도 되며, 그 결과 리스트 전체 길이가 늘어나거나 줄어들 수 있다.
- 빈 리스트를 대입하면 그 구간의 항목들을 통째로 삭제하는 효과가 난다.
- `리스트[:] = []`처럼 전체 슬라이스에 빈 리스트를 대입하면 리스트를 완전히 비울 수 있다.

```python
letters = ['a', 'b', 'c', 'd', 'e']
letters[1:3] = ['B', 'C', 'X']   # 두 자리를 세 값으로 바꾼다
letters   # ['a', 'B', 'C', 'X', 'd', 'e'] -- 길이가 5에서 6으로 늘어난다

letters[1:4] = []                # 세 자리를 통째로 삭제한다
letters   # ['a', 'd', 'e']

letters[:] = []                  # 전체를 비운다
letters   # []
```

### 6. `len()`은 리스트에도 그대로 적용된다

문자열 길이를 잴 때 쓰던 `len()`을 리스트에도 똑같이 쓸 수 있다. 리스트 안에 들어 있는 항목의 개수를 돌려준다.

```python
fruits = ['apple', 'banana', 'grape']
len(fruits)  # 3
```

### 7. 리스트 안에 리스트를 넣으면(중첩 리스트) 인덱스를 연달아 붙여서 접근한다

리스트는 다른 리스트를 항목으로 담을 수 있다. 이렇게 만든 중첩 리스트는 인덱스를 두 번 이어서 써서 안쪽 값에 접근한다.

```python
row1 = ['x', 'y', 'z']
row2 = [10, 20, 30]
grid = [row1, row2]

grid[0]      # ['x', 'y', 'z']
grid[0][1]   # 'y'
grid[1][2]   # 30
```

### 8. 다중 대입은 우변을 먼저 전부 계산한 뒤에 한꺼번에 대입한다

`a, b = 0, 1`처럼 여러 변수에 동시에 값을 대입할 수 있다. 이때 중요한 점은, 오른쪽에 있는 값들을 먼저 왼쪽에서 오른쪽 순서로 전부 계산한 다음, 그 결과를 한 번에 왼쪽 변수들에 대입한다는 것이다. 그래서 `a, b = b, a + b`처럼 서로를 참조하는 대입도 문제없이 동작한다(먼저 b와 a+b를 계산해두고, 그 다음에 대입하기 때문이다).

```python
a, b = 0, 1
a, b = b, a + b   # 오른쪽 (b, a+b) = (1, 1)을 먼저 계산한 뒤 대입한다
a, b              # (1, 1)
```

### 9. `while` 문의 조건에는 숫자뿐 아니라 문자열·리스트도 그대로 쓸 수 있다

- `while` 조건이 참인 동안 반복이 계속된다.
- 파이썬에서는 0이 아닌 모든 정수는 참으로, 0은 거짓으로 취급된다.
- 문자열이나 리스트 같은 시퀀스도 조건 자리에 바로 쓸 수 있는데, 길이가 0이 아니면 참, 빈 시퀀스는 거짓으로 취급된다.

```python
pendingTasks = ['이메일 확인', '보고서 작성']

while pendingTasks:
    task = pendingTasks.pop(0)
    print(task, '처리 완료')
```

`pendingTasks`가 빈 리스트가 되는 순간 조건이 거짓이 되어 반복이 자동으로 끝난다.

### 10. 들여쓰기가 곧 블록을 구분하는 문법이다

- 파이썬은 중괄호 대신 들여쓰기로 코드 블록을 구분한다.
- 같은 블록에 속하는 줄은 반드시 같은 만큼 들여써야 한다. 들여쓰기 양이 줄마다 다르면 에러가 난다.

## 11. `print()`는 여러 인자를 한 번에 출력하고, `end`로 줄바꿈 방식을 바꿀 수 있다

- `print()`에 여러 값을 쉼표로 나열하면, 값 사이에 자동으로 빈칸이 들어가면서 한 줄로 출력된다. 문자열은 따옴표 없이 출력된다.
- 기본적으로 `print()`는 끝에 줄바꿈(개행)을 추가한다. `end` 키워드 인자를 주면 그 줄바꿈 대신 원하는 문자열로 끝맺을 수 있다.

```python
count = 3
print('현재 카운트는', count, '입니다')  # 현재 카운트는 3 입니다

for number in range(5):
    print(number, end=' | ')
# 0 | 1 | 2 | 3 | 4 |
```

### 오늘의 한 줄 정리

리스트를 다룰 때 가장 조심해야 할 부분은 "대입은 복사가 아니다"라는 점이다. 원본을 보존하고 싶으면 슬라이스(`[:]`)로 새 리스트를 만들어야 하고, 그렇지 않으면 한 변수에서의 변경이 같은 리스트를 가리키는 다른 모든 변수에도 그대로 나타난다.
