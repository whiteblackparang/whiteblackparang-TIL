# 오늘 배운 점

## 서버(Server)와 클라이언트(Client)

웹에서는 클라이언트가 요청(Request)을 보내고, 서버가 응답(Response)을 보내는 방식으로 동작한다.

- **클라이언트(Client)** : 서비스를 사용하는 사용자(브라우저, 앱 등)
- **서버(Server)** : 요청을 받아 데이터를 처리하고 결과를 반환하는 컴퓨터

예를 들어 사용자가 로그인 버튼을 누르면 브라우저가 서버에게 로그인 요청을 보내고, 서버는 로그인 결과를 다시 브라우저에게 전달한다.

---

## Web API

웹에서는 서버와 클라이언트가 데이터를 주고받기 위해 API(Application Programming Interface)를 사용한다.

API는 서버가 요청을 받고 응답하기 위해 미리 정해 놓은 규칙(명세)이다.

클라이언트는 API 주소(URL)로 요청을 보내고, 서버는 요청에 맞는 데이터를 응답한다.

API를 사용하면 HTML뿐만 아니라 JSON과 같은 데이터 형식으로도 정보를 주고받을 수 있다.

---

## JSON

JSON(JavaScript Object Notation)은 JavaScript에서 파생된 데이터 표현 방식이다.

서버와 클라이언트가 데이터를 주고받을 때 가장 많이 사용하는 형식이며, 사람이 읽기 쉽고 용량이 비교적 가벼운 텍스트 기반의 데이터 포맷이다.

데이터는 키(Key)와 값(Value)형태로 저장된다.

---

## HTTP 요청(Request)

클라이언트가 서버에 요청을 보낼 때는 요청의 목적에 따라 여러 가지 메서드를 사용한다.

### GET

데이터를 조회할 때 사용하는 요청 방식이다.

주로 서버의 데이터를 가져올 때 사용하며, 서버의 데이터를 변경하지 않는다.

---

### POST

새로운 데이터를 생성하거나 서버에 데이터를 전달할 때 사용하는 요청 방식이다.

회원가입, 로그인, 게시글 작성 등에 많이 사용된다.

※ 실제로 데이터 수정(PUT, PATCH)이나 삭제(DELETE)는 별도의 메서드를 사용하지만, 간단한 프로젝트에서는 POST로 처리하는 경우도 있다.

---

## 데이터 전달 방법(Query String)

GET 요청에서는 URL 뒤에 데이터를 함께 전달할 수 있다.

- `?` : 전달할 데이터가 시작된다는 의미
- `&` : 전달할 데이터가 더 있다는 의미
- `이름=값` 형태로 데이터를 전달한다.

URL에서 사용하는 데이터 이름(매개변수 이름)은 프론트엔드와 백엔드 개발자가 미리 약속하여 사용한다.
