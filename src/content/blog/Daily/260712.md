# 오늘 배운 점

파이썬 공식 문서 4장 뒷부분(함수 정의)을 읽고 정리한다. 함수 정의 기초부터 기본 인자값의 함정, 위치 전용/키워드 전용 매개변수, 가변 인자, 언패킹, 람다, 독스트링, 함수 어노테이션까지 다룬다.

## 1. 함수는 이름과 함수 객체를 연결할 뿐이다

- `def`로 함수를 정의하면, 그 함수 이름은 심볼 테이블에서 함수 객체를 가리키게 된다. 그래서 같은 함수 객체를 다른 이름에도 대입할 수 있고, 그 새 이름으로도 똑같이 호출할 수 있다.
- `return` 문이 없는 함수도 값을 돌려준다. 다만 그 값이 `None`일 뿐이다. 함수 본문이 끝까지 실행되고도 `return`을 안 만나면 자동으로 `None`이 반환된다.

```python
def logMessage(text):
    print('[LOG]', text)

alias = logMessage
alias('서버 시작')       # logMessage와 완전히 동일하게 동작

result = logMessage('완료')
print(result)            # None, return 문이 없었기 때문
```

## 2. 기본 인자값은 함수가 정의되는 시점에 딱 한 번만 계산된다

- 매개변수에 기본값을 지정하면, 호출할 때 그 인자를 생략해도 된다.
- 중요한 함정: 기본값이 리스트나 딕셔너리처럼 가변 객체라면, 그 객체는 함수가 처음 정의될 때 딱 한 번만 만들어지고, 이후 모든 호출에서 계속 재사용된다. 그래서 이전 호출에서 남긴 변경 내용이 다음 호출에도 그대로 남아있는 문제가 생긴다.
- 해결책은 기본값을 `None`으로 두고, 함수 본문에서 `None`일 때만 새 리스트(또는 딕셔너리)를 만드는 것이다.

```python
def addLogWrong(entry, history=[]):
    history.append(entry)
    return history

addLogWrong('첫 번째')   # ['첫 번째']
addLogWrong('두 번째')   # ['첫 번째', '두 번째'] -- 이전 호출 결과가 남아있다

def addLogRight(entry, history=None):
    if history is None:
        history = []
    history.append(entry)
    return history

addLogRight('첫 번째')   # ['첫 번째']
addLogRight('두 번째')   # ['두 번째'] -- 매번 새로 시작한다
```

## 3. 키워드 인자로 호출하면 순서보다 이름이 우선한다

- 함수를 호출할 때 `이름=값` 형태로 인자를 넘기면, 인자를 나열하는 순서가 매개변수 정의 순서와 달라도 상관없다.
- 단, 키워드 인자는 반드시 위치 인자 뒤에 와야 하고, 같은 인자에 값을 두 번 줄 수는 없다.

```python
def bookTicket(destination, seatClass='economy', count=1):
    return f'{destination}행 {seatClass} {count}석'

bookTicket('부산')                              # 위치 인자 하나만
bookTicket('부산', count=2)                     # 위치 + 키워드
bookTicket(count=3, destination='제주', seatClass='business')  # 순서를 바꿔도 된다
```

## 4. `/`와 `*`로 인자 전달 방식을 강제할 수 있다

- 매개변수 목록에서 `/` 앞에 있는 매개변수는 위치로만 전달할 수 있다(이름으로는 전달 불가).
- `*` 뒤에 있는 매개변수는 키워드로만 전달할 수 있다(위치로는 전달 불가).
- `/`와 `*` 사이(또는 둘 다 없을 때 전체)는 위치·키워드 어느 쪽으로도 전달할 수 있다.

```python
def transfer(fromAccount, toAccount, /, amount, *, memo):
    print(fromAccount, '->', toAccount, amount, memo)

transfer('A001', 'A002', amount=5000, memo='정산')  # 올바른 호출
# transfer(fromAccount='A001', toAccount='A002', amount=5000, memo='정산')
# 위 호출은 에러가 난다. fromAccount, toAccount는 위치 전용이라 이름으로 못 넘긴다.
```

## 5. `*args`로 개수가 정해지지 않은 위치 인자를 한 번에 받는다

- 매개변수 앞에 `*`를 붙이면, 그 자리부터 남은 위치 인자들을 모두 튜플 하나로 묶어서 받는다.
- `*args` 뒤에 오는 매개변수는 위치로 전달할 방법이 없으므로 자동으로 키워드 전용이 된다.

```python
def total(label, *amounts):
    return label, sum(amounts)

total('합계', 1000, 2000, 3000)  # ('합계', 6000)
```

## 6. `**kwargs`로 정해지지 않은 키워드 인자를 딕셔너리로 받는다

- 매개변수 앞에 `**`를 붙이면, 미리 정의되지 않은 키워드 인자들을 이름-값 쌍이 담긴 딕셔너리 하나로 받는다.
- `*args`와 `**kwargs`는 함께 쓸 수 있는데, 이때 `*args`가 반드시 `**kwargs`보다 앞에 와야 한다.

```python
def createProfile(username, *hobbies, **extra):
    print('아이디:', username)
    print('취미:', hobbies)
    for key, value in extra.items():
        print(key, ':', value)

createProfile('jihoon', '독서', '등산', city='서울', age=28)
```

## 7. 리스트나 딕셔너리를 `*`, `**`로 풀어서 인자로 전달할 수 있다(언패킹)

- 이미 리스트나 튜플에 담긴 값을 개별 위치 인자로 넘기고 싶다면, 호출할 때 그 앞에 `*`를 붙인다.
- 딕셔너리를 키워드 인자로 통째로 넘기고 싶다면 `**`를 붙인다.

```python
def createRange(start, stop, step):
    return list(range(start, stop, step))

args = [0, 10, 2]
createRange(*args)   # createRange(0, 10, 2)와 동일

def bookTicket(destination, seatClass='economy', count=1):
    return f'{destination}행 {seatClass} {count}석'

options = {'destination': '제주', 'seatClass': 'business', 'count': 2}
bookTicket(**options)
```

## 8. `lambda`로 이름 없는 짧은 함수를 즉석에서 만들 수 있다

- `lambda 매개변수들: 표현식` 형태로, 이름이 없는 함수를 한 줄로 만들 수 있다. 문법상 표현식 하나로만 제한되며, 의미상으로는 일반 `def` 함수 정의를 짧게 줄인 것과 같다.
- 함수를 인자로 받는 자리(예: 정렬 기준을 지정하는 `key`)에 짧은 함수가 필요할 때 자주 쓴다.
- `lambda`로 만든 함수도 자신을 둘러싼 스코프의 변수를 그대로 참조할 수 있다.

```python
def makeMultiplier(factor):
    return lambda x: x * factor

double = makeMultiplier(2)
double(5)   # 10

students = [('민호', 82), ('유나', 95), ('도윤', 77)]
students.sort(key=lambda student: student[1])
students    # [('도윤', 77), ('민호', 82), ('유나', 95)]
```

## 9. 독스트링은 첫 줄 요약 + 빈 줄 + 상세 설명 순서로 쓰는 관례가 있다

- 함수 본문의 첫 문장이 문자열 리터럴이면 그 함수의 독스트링(설명 문서)이 된다.
- 관례상 첫 줄은 함수의 목적을 짧게 요약하고 마침표로 끝내며, 함수 이름이나 타입은 굳이 다시 언급하지 않는다.
- 설명이 여러 줄이면, 요약 다음 줄을 비워서 요약과 상세 설명을 시각적으로 분리한다.

```python
def calculateDiscount(price, rate):
    """정가에서 할인율을 적용한 최종 가격을 계산한다.

    rate는 0에서 1 사이의 소수로 전달해야 한다.
    예를 들어 10% 할인이면 rate=0.1을 전달한다.
    """
    return price * (1 - rate)

print(calculateDiscount.__doc__)
```

## 10. 함수 어노테이션은 타입에 대한 참고 정보일 뿐, 강제로 검사하지는 않는다

- 매개변수 뒤에 `: 타입`을 붙이고, 반환값 앞에 `-> 타입`을 붙이면 함수 어노테이션이 된다.
- 이 정보는 함수의 `__annotations__` 딕셔너리에 저장될 뿐, 파이썬이 실제로 타입을 강제하지는 않는다. 코드를 읽는 사람이나 별도의 타입 검사 도구를 위한 참고 정보에 가깝다.

```python
def buildGreeting(name: str, times: int = 1) -> str:
    return (name + '님 안녕하세요! ') * times

buildGreeting.__annotations__  # {'name': str, 'times': int, 'return': str}
buildGreeting('수진', 2)        # 어노테이션과 다른 타입을 넣어도 에러 없이 그대로 실행
```
