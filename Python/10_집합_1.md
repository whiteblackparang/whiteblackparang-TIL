# 튜플 (Tuple)

## 튜플이란?

튜플(Tuple)은 **변경이 불가능한 순서 있는 자료형**.

- 소괄호 `()` 사용
- 요소의 순서 유지
- 인덱스로 요소 접근 가능
- 생성 후 요소 추가·삭제·수정 불가
- 변경되면 안 되는 데이터 저장에 적합
- 딕셔너리의 키로 사용 가능

---

## 다양한 튜플 생성

튜플을 만드는 여러 방법.

- 소괄호 `()` 사용
- 쉼표(`,`)만으로 생성 가능
- 숫자, 문자열, 불리언 등 다양한 자료형 저장 가능
- 하나의 요소를 가진 튜플은 쉼표 필수
- 빈 튜플은 `()` 또는 `tuple()` 사용

---

## 튜플 인덱싱

인덱스로 요소 접근.

- 양수 인덱스 사용 가능
- 음수 인덱스 사용 가능
- 리스트와 동일한 인덱싱 방식
- 요소 조회 전용

---

## 튜플 슬라이싱

원하는 범위의 요소 추출.

- `시작:끝` 문법 사용
- `step` 지정 가능
- 역순 슬라이싱 가능
- 새로운 튜플 반환
- 원본 튜플 유지

---

## 튜플의 불변성

생성 후 내용 변경 불가.

- 요소 수정 불가
- 요소 추가 불가
- 요소 삭제 불가
- 인덱스를 이용한 변경 불가
- 안전한 데이터 보관에 적합

---

## 튜플 연결과 반복

새로운 튜플 생성.

- `+` 연산자로 연결
- `*` 연산자로 반복
- 원본 튜플 유지
- 새로운 튜플 반환

---

## in / not in 연산자

요소 포함 여부 확인.

- `in` : 요소 존재 여부 확인
- `not in` : 요소 미포함 여부 확인
- 반환값은 `True` 또는 `False`
- 리스트와 동일한 사용 방법

---

## 튜플 패킹

여러 값을 하나의 튜플로 묶기.

- 쉼표(`,`)만으로 튜플 생성
- 소괄호 생략 가능
- 여러 값 자동 묶음
- 함수의 여러 반환값 표현에 활용

---

## 튜플 언패킹

튜플을 여러 변수로 분리.

- 요소 개수와 변수 개수 일치 필요
- 여러 변수에 한 번에 값 저장
- 값 교환에 자주 활용
- 함수 반환값 분리에 활용

---

## 튜플과 리스트 변환

자료형 상호 변환.

- `tuple()` : 리스트 → 튜플
- `list()` : 튜플 → 리스트
- 문자열도 변환 가능
- 수정이 필요하면 리스트 사용
- 변경 방지가 필요하면 튜플 사용

---

## 튜플 메서드

튜플에서 사용할 수 있는 메서드.

### count()

- 특정 요소의 개수 반환

### index()

- 특정 요소의 첫 번째 위치 반환

### 특징

- 수정 메서드 제공 없음
- `append()`, `remove()` 등 사용 불가

---

## 튜플 길이

튜플의 요소 개수 확인.

- `len()` 함수 사용
- 전체 요소 개수 반환
- 리스트, 문자열과 동일한 사용 방법

---

## 핵심 정리

| 기능 | 사용 방법 |
|------|-----------|
| 튜플 생성 | `()` |
| 빈 튜플 생성 | `()` 또는 `tuple()` |
| 요소 접근 | `[]` |
| 슬라이싱 | `[:]` |
| 연결 | `+` |
| 반복 | `*` |
| 포함 여부 확인 | `in`, `not in` |
| 리스트 변환 | `list()` |
| 튜플 변환 | `tuple()` |
| 요소 개수 | `count()` |
| 요소 위치 | `index()` |
| 길이 확인 | `len()` |

> **참고**
>
> 튜플은 생성 이후 내용을 변경할 수 없는 불변(Immutable) 자료형.
