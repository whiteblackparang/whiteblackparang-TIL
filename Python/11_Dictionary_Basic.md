# 딕셔너리 기초

## 딕셔너리란?

**키(Key)와 값(Value)을 쌍으로 저장하는 자료형**

- 중괄호 `{}` 사용
- `키: 값` 형태로 데이터 저장
- 키를 이용한 빠른 값 조회
- 키는 중복 불가
- 값은 중복 가능
- Python 3.7부터 입력 순서 유지

---

## 딕셔너리 생성하기

딕셔너리를 만드는 다양한 방법.

- `{}`로 직접 생성
- `dict()` 함수 사용 가능
- 빈 딕셔너리는 `{}` 또는 `dict()`
- 키는 변경 불가능(Immutable) 자료형 사용
- 값은 모든 자료형 저장 가능

---

## 키로 값 접근하기

키를 이용한 데이터 조회.

- `dict[key]` 형식 사용
- 키에 해당하는 값 반환
- 존재하지 않는 키 접근 시 `KeyError` 발생
- 키 이름의 대소문자 구분

---

## 키로 값 수정하기

기존 키의 값 변경.

- `dict[key] = value` 형식 사용
- 기존 값 덮어쓰기
- 원본 딕셔너리 변경
- 같은 키에 새로운 값 저장

---

## 새 키-값 쌍 추가하기

새로운 데이터 저장.

- 존재하지 않는 키에 값 할당
- 새로운 키-값 쌍 자동 추가
- 기존 키면 수정
- 새로운 키면 추가

---

## del로 키-값 삭제

특정 항목 삭제.

- `del dict[key]` 사용
- 키와 값 함께 삭제
- 존재하지 않는 키 삭제 시 `KeyError` 발생
- 원본 딕셔너리 변경

---

## in / not in 연산자

키 존재 여부 확인.

- `in` : 키 존재 여부 확인
- `not in` : 키 미존재 여부 확인
- 반환값은 `True` 또는 `False`
- 값이 아닌 키 기준 검사

---

## 딕셔너리 길이

저장된 항목 개수 확인.

- `len()` 함수 사용
- 키-값 쌍의 개수 반환
- 키의 개수와 동일
- 리스트, 튜플, 집합과 동일한 사용 방법

---

## 여러 키-값 접근

필요한 데이터 선택.

- 여러 키를 개별적으로 조회
- 원하는 데이터만 추출 가능
- 새로운 딕셔너리 생성 가능
- 슬라이싱 기능 제공 없음

---

## 중첩 딕셔너리

딕셔너리 안에 딕셔너리 저장.

- 값으로 딕셔너리 저장 가능
- 계층 구조 표현
- 연속된 대괄호로 접근
- JSON 구조와 유사

---

## 다양한 키 타입

여러 자료형을 키로 사용.

- 문자열 사용 가능
- 숫자 사용 가능
- 튜플 사용 가능
- 변경 가능한 자료형 사용 불가
- `frozenset`은 키 사용 가능

---

## 핵심 정리

| 기능 | 사용 방법 |
|------|-----------|
| 딕셔너리 생성 | `{}` 또는 `dict()` |
| 빈 딕셔너리 생성 | `{}` 또는 `dict()` |
| 값 조회 | `dict[key]` |
| 값 수정 | `dict[key] = value` |
| 항목 추가 | `dict[new_key] = value` |
| 항목 삭제 | `del dict[key]` |
| 키 존재 확인 | `in`, `not in` |
| 항목 개수 확인 | `len()` |

> **참고**
>
> 딕셔너리는 키를 기준으로 데이터를 저장하는 자료형. 키는 중복될 수 없으며, 하나의 키는 하나의 값만 가짐.
