# NULL이란?

`NULL`은 **값이 존재하지 않는 상태**를 의미한다.

0(숫자), 공백(`''`), 빈 문자열과는 다른 개념이며, "아직 값이 입력되지 않았다" 또는 "알 수 없는 값"을 나타낸다.

NULL은 일반적인 값이 아니기 때문에 계산이나 비교 연산에서도 특별한 규칙이 적용된다.

예를 들어 숫자에 NULL을 더하면 결과 역시 NULL이 된다.

즉,

```text
100 + NULL = NULL
```

과 같이 계산된다.

따라서 NULL이 포함된 데이터를 그대로 계산하면 원하는 결과를 얻지 못하는 경우가 많다.

---

# NVL

`NVL`은 Oracle에서 가장 많이 사용하는 **결측치 처리 함수**이다.

컬럼의 값이 NULL인 경우 사용자가 지정한 다른 값으로 변경하여 반환한다.

기본 문법은 다음과 같다.

```sql
NVL(컬럼, 대체값)
```

동작 방식은 매우 단순하다.

- 값이 NULL이면 **대체값 반환**
- NULL이 아니면 **원래 값 반환**

즉, NULL을 다른 값으로 치환하는 함수라고 이해하면 된다.

실무에서는 다음과 같은 상황에서 자주 사용된다.

- NULL을 0으로 변경하여 계산
- NULL을 빈 문자열로 출력
- NULL 대신 기본값을 표시

특히 급여, 수당, 할인금액처럼 계산이 필요한 컬럼에서는 거의 필수적으로 사용되는 함수이다.

---

# NVL2

`NVL2`는 `NVL`을 확장한 함수이다.

`NVL`은 NULL일 때만 대체값을 지정할 수 있지만,

`NVL2`는 **NULL이 아닌 경우와 NULL인 경우 각각 다른 값을 반환**할 수 있다.

기본 문법은 다음과 같다.

```sql
NVL2(컬럼, NULL이 아닐 때 값, NULL일 때 값)
```

동작 방식은 다음과 같다.

- 값이 존재하면 첫 번째 값을 반환
- 값이 NULL이면 두 번째 값을 반환

즉, 단순히 NULL을 대체하는 것이 아니라 **NULL 여부를 조건처럼 판단**하여 서로 다른 결과를 반환하는 함수이다.

실무에서는 다음과 같은 경우에 많이 사용된다.

- 보너스 지급 여부 표시
- 데이터 존재 여부 표시
- 조건에 따라 다른 계산 수행
- 상태값 출력

---

# NVL과 NVL2의 차이

두 함수 모두 NULL을 처리하기 위한 함수이지만 목적에는 차이가 있다.

|함수|특징|
|:---|:---|
|NVL|NULL을 하나의 값으로 변경|
|NVL2|NULL 여부에 따라 서로 다른 값을 반환|

간단하게 기억하면 다음과 같다.

- **NVL** : "NULL이면 무엇으로 바꿀까?"
- **NVL2** : "NULL인지 아닌지에 따라 어떤 결과를 보여줄까?"

---

# 언제 사용할까?

NULL은 데이터베이스에서 매우 흔하게 등장한다.

예를 들어

- 추가 수당이 없는 직원
- 아직 입력되지 않은 연락처
- 배송되지 않은 주문
- 입력 대기 중인 날짜

등은 대부분 NULL로 저장된다.

이러한 데이터를 그대로 계산하거나 출력하면 원하는 결과를 얻기 어렵기 때문에, 상황에 맞게 `NVL` 또는 `NVL2`를 활용하여 결측치를 처리하는 것이 중요하다.

---

# 마무리

`NULL`은 값이 없는 상태를 의미하며, 일반적인 데이터와는 다른 방식으로 처리된다.

Oracle에서는 이러한 결측치를 처리하기 위해 `NVL`과 `NVL2` 함수를 제공한다.

- **NVL**은 NULL을 지정한 값으로 대체하는 함수이다.
- **NVL2**는 NULL인지 여부에 따라 서로 다른 값을 반환하는 함수이다.

결측치 처리는 SQL에서 매우 기본적이면서도 중요한 개념이다. 데이터를 정확하게 계산하고 원하는 형태로 출력하기 위해서는 NULL의 특성을 이해하고 `NVL`, `NVL2`를 적절히 활용하는 것이 중요하다.
