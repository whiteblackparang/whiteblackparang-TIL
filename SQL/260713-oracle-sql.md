# 오라클 SQL 기초 : 트랜잭션과 세션

## 트랜잭션(Transaction)

더 이상 나눌 수 없는 하나의 작업 단위입니다.

예를 들어 계좌 이체는 출금과 입금이 모두 성공해야 하나의 트랜잭션이
됩니다.

## TCL

-   COMMIT : 변경사항 저장
-   ROLLBACK : 마지막 COMMIT 이후 변경 취소

## 세션(Session)

로그인부터 로그아웃까지의 연결입니다. 하나의 세션 안에서 여러 트랜잭션이
수행됩니다.

## 읽기 일관성

다른 세션이 아직 COMMIT하지 않은 변경 내용은 보이지 않고, 마지막으로
확정된 데이터를 조회합니다.

## LOCK

데이터를 수정하는 동안 다른 세션의 수정을 막는 기능입니다. COMMIT 또는
ROLLBACK이 수행되면 LOCK이 해제됩니다.

# 핵심 정리

-   트랜잭션은 하나의 작업 단위이다.
-   COMMIT은 저장, ROLLBACK은 취소이다.
-   세션은 로그인부터 로그아웃까지의 연결이다.
-   읽기 일관성으로 안정적인 조회가 가능하다.
-   LOCK은 동시에 같은 데이터를 수정하지 못하도록 한다.
