# 오라클 SQL 기초 : 결측치 처리(NVL, NVL2)

데이터를 조회하거나 계산하다 보면 **NULL(결측치)** 을 자주 만나게 된다.

Oracle에서 `NULL`은 **값이 없는 상태**를 의미하며, 숫자 0이나 빈 문자열과는 다른 개념이다. 특히 NULL이 포함된 상태에서 계산을 수행하면 예상과 다른 결과가 발생할 수 있으므로 적절한 처리가 필요하다.

Oracle에서는 이러한 문제를 해결하기 위해 **NVL**과 **NVL2** 함수를 제공한다.

이번 글에서 다루는 내용은 다음과 같다.

- NULL이란?
- NVL
- NVL2

---

# 1. NULL이란?

`NULL`은 데이터가 **존재하지 않는 상태**를 의미한다.

예를 들어 EMP 테이블의 `COMM`(추가 수당) 컬럼은 일부 직원만 값을 가지고 있으며, 지급되지 않은 직원은 `NULL`로 저장되어 있다.

```sql
SELECT COMM
FROM EMP;
```

실행 결과를 보면 일부 행은 값이 존재하지만, 일부는 NULL로 표시된다.

이러한 NULL은 계산에도 영향을 준다.

```sql
SELECT EMPNO,
       ENAME,
       SAL + COMM
FROM EMP;
```

급여(`SAL`)와 추가 수당(`COMM`)을 더하려고 했지만, `COMM`이 NULL인 직원은 계산 결과도 NULL이 된다.

이는 Oracle에서 NULL과의 연산 결과는 항상 NULL이 되기 때문이다.

---

# 2. NVL

`NVL`은 NULL 값을 원하는 값으로 변경하는 함수이다.

기본 문법은 다음과 같다.

```sql
NVL(컬럼, 대체값)
```

- 컬럼이 NULL이면 **대체값**을 반환한다.
- NULL이 아니면 원래 값을 그대로 반환한다.

예를 들어 추가 수당이 없는 직원은 0으로 처리하여 계산하려면 다음과 같이 작성한다.

```sql
SELECT EMPNO,
       ENAME,
       SAL + NVL(COMM, 0)
FROM EMP;
```

`COMM`이 NULL인 경우에는 0으로 변경한 뒤 계산하므로 급여만 정상적으로 출력된다.

반대로 `COMM`에 값이 있는 경우에는 기존 값을 그대로 사용한다.

---

## NVL 사용 예시

다음 SQL은 NULL을 0으로 변경하여 출력한다.

```sql
SELECT COMM,
       NVL(COMM, 0)
FROM EMP;
```

예를 들어 결과가 다음과 같다면

|COMM|NVL(COMM, 0)|
|:---:|:---:|
|300|300|
|500|500|
|NULL|0|
|NULL|0|

NULL인 값만 0으로 변경되는 것을 확인할 수 있다.

실무에서는 급여, 수당, 할인 금액처럼 NULL이 포함될 수 있는 데이터를 계산할 때 가장 많이 사용하는 함수 중 하나이다.

---

# 3. NVL2

`NVL2`는 `NVL`을 확장한 함수이다.

`NVL`은 NULL일 때의 값만 지정할 수 있지만, `NVL2`는 **NULL이 아닐 때와 NULL일 때 각각 다른 값을 지정**할 수 있다.

기본 문법은 다음과 같다.

```sql
NVL2(컬럼, NULL이 아닐 때 값, NULL일 때 값)
```

예를 들어 추가 수당(COMM)의 존재 여부에 따라 다른 문구를 출력할 수 있다.

```sql
SELECT COMM,
       NVL(COMM, 0),
       NVL2(COMM, '보너스 있음', '보너스 없음')
FROM EMP;
```

결과는 다음과 같은 형태로 출력된다.

|COMM|NVL|NVL2|
|:---:|:---:|:---:|
|300|300|보너스 있음|
|500|500|보너스 있음|
|NULL|0|보너스 없음|

`NVL2`는 단순히 NULL을 대체하는 것이 아니라, NULL 여부에 따라 서로 다른 결과를 반환할 수 있다는 점이 특징이다.

---

## 연봉 계산 예시

다음 SQL은 추가 수당이 있는 직원과 없는 직원의 연봉을 각각 계산한다.

```sql
SELECT EMPNO,
       ENAME,
       COMM,
       NVL2(COMM,
            SAL * 12 + COMM,
            SAL * 12) AS ANNSAL
FROM EMP;
```

동작 방식은 다음과 같다.

- `COMM`이 존재하면 `SAL × 12 + COMM`
- `COMM`이 NULL이면 `SAL × 12`

즉, 추가 수당이 있는 직원은 수당을 포함한 연봉을 계산하고, 없는 직원은 기본 연봉만 계산한다.

이처럼 `NVL2`는 조건에 따라 서로 다른 계산식을 적용해야 하는 경우 매우 유용하다.

---

# NVL과 NVL2 비교

|함수|설명|
|:---|:---|
|NVL(컬럼, 대체값)|NULL이면 대체값 반환|
|NVL2(컬럼, NULL 아님, NULL)|NULL 여부에 따라 서로 다른 값 반환|

간단히 정리하면,

- **NVL**은 "NULL이면 무엇으로 바꿀까?"
- **NVL2**는 "NULL인지 아닌지에 따라 어떤 값을 반환할까?"

를 결정하는 함수라고 이해하면 쉽다.

---

# 마무리

이번 글에서는 Oracle의 대표적인 결측치 처리 함수인 `NVL`과 `NVL2`를 정리했다.

- **NULL**은 값이 없는 상태를 의미하며, 계산 시 결과도 NULL이 된다.
- **NVL**은 NULL을 원하는 값으로 대체할 때 사용한다.
- **NVL2**는 NULL 여부에 따라 서로 다른 값을 반환할 수 있다.

결측치 처리는 데이터 분석과 SQL 작성에서 매우 중요한 개념이다. 특히 급여나 수당처럼 NULL이 포함될 수 있는 데이터를 계산할 때 `NVL`과 `NVL2`를 적절히 활용하면 보다 정확한 결과를 얻을 수 있다.
